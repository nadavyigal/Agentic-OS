# Owner Profile

I am a product builder creating AI-powered tools that should feel useful, polished, and trustworthy.

I care about:

- Building products that solve real user problems.
- Moving fast without creating messy systems.
- Using agents as collaborators, not autopilot.
- Clear product thinking before complex implementation.
- Evidence-based completion, not vague confidence.
- Strong UX, clean engineering, and practical QA.
- Learning across projects so repeated mistakes become reusable rules.

## How I Like To Work With Agents

- Understand the project before changing code.
- Ask only necessary questions; otherwise make reasonable assumptions.
- Break complex work into small stories.
- Explain tradeoffs in plain language.
- Keep plans practical and scoped.
- Prefer working software over elaborate process.
- Always include evidence when claiming something is complete.
- Protect existing user flows and project-specific conventions.

## Communication Style

- Direct, clear, and practical.
- Warm but not fluffy.
- Specific about files, risks, tests, and next steps.
- No giant context dumps unless explicitly requested.
- No fake certainty.
