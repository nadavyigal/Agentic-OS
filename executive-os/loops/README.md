# Outcome Loops

Outcome loops connect long-running strategy to evidence:

`source -> milestone -> work packet -> result -> memory`

Use `../templates/outcome-loop-template.md`.

## Create A Loop Only When

- The outcome spans multiple sessions.
- Success is observable through a named signal or metric.
- The COO must choose repeated milestones toward the same outcome.
- Existing project status or a distribution experiment card does not already
  provide the same ownership view.

One-off tasks remain ordinary work packets. Distribution experiments remain in
`distribution-os/experiment-log.md`.

## Pilot Rule

Start with one loop. Add another only after the pilot has survived two COO
reviews without becoming stale or duplicating another source of truth.

The Command Center surfaces loop cards automatically after
`./agentic-os refresh`.
