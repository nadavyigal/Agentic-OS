# AGENTS.md - Executive OS

Rules for agents working in `executive-os/`. The root `AGENTS.md` still applies.

## Start Here

Before strategy, finance, research, or weekly/monthly review work:

1. Read `executive-os/README.md`.
2. Read `executive-os/EXECUTIVE-DECISIONS.md`.
3. Read `executive-os/EXECUTIVE-METRICS.md`.
4. Read the relevant workflow in `executive-os/workflows/`.
5. State the objective and identify the evidence sources.

## Executive Rules

- Separate facts from assumptions.
- Do not invent financial, activation, revenue, cost, conversion, or user numbers.
- Use `unknown - need: <source>` when a number is missing.
- Cite sources for research claims, or label the source type and reliability.
- CEO outputs must recommend a decision, not only summarize.
- CFO and monetization outputs must include risks and a validation plan.
- Analysis outputs must end with confidence level and one recommended next step.

## Stop And Ask Before

- Changing OKRs, executive cadence, financial assumptions, or durable decisions.
- Treating stale dashboards as current without refreshing source files.
- Recommending spend, pricing, monetization, or launch commitments without evidence.
- Pulling product implementation work into this folder instead of the product repo.
