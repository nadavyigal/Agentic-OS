## Key Themes Across All Articles

1. **CLAUDE.md as persistent memory** — The single highest-leverage setup action. One file read automatically every session.
2. **Projects + Custom Instructions** — Separate brains for separate work, each with targeted context.
3. **Context engineering > prompt engineering** — The environment beats the syntax every time.
4. **Memory systems** — MEMORY.md for decisions, ERRORS.md for failed approaches, session summaries.
5. **The .claude folder as infrastructure** — Rules, hooks, agents, permissions — not just instructions.
6. **Karpathy's 4 rules** — Ask don't assume, simplest first, don't touch unrelated code, flag uncertainty.
7. **Modular rules over monolithic files** — Scoped, maintainable, context-efficient.
8. **Hooks for safety** — Intercept tool calls before/after to enforce guardrails.
9. **Specialized subagents** — Focused agents with limited tools beat one giant general-purpose agent.
10. **MCP for tool integration** — Context + tools = the model becomes an operator, not just an advisor.
