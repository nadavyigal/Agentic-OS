# Distribution Lessons

Append-only. One lesson per row. Cross-product. Project-specific lessons live in each project's `.agent-os/distribution/lessons.md`.

Capture rule: a lesson goes here only if it has happened twice, costs real time when forgotten, or is non-obvious enough that a future agent would miss it.

## How To Add A Lesson

1. Confirm it has happened twice or cost >= 30 minutes to relearn
2. Write it as a rule a future agent can apply
3. Cite evidence (experiment ID, week, file)
4. If it is cross-product, also append to `~/.claude/LEARNINGS.md`
5. If a failed pattern, add to `~/.claude/ERRORS.md` as well

## Promotion Rules

- Twice = lesson here
- Three times across products = promote to `~/.claude/LEARNINGS.md`
- Discovered during a workflow = link the lesson row from the workflow file so the workflow itself improves
